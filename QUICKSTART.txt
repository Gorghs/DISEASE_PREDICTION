──────────────────────────────────────

→ Click "New +" → "Web Service"
→ Connect GitHub repo
→ Build Command: pip install -r requirements.txt && python train.py
→ Start Command: gunicorn --workers 1 --timeout 120 --bind 0.0.0.0:$PORT app:app
→ Add Environment Variables:
   • KAGGLE_USERNAME (your kaggle username)
   • KAGGLE_KEY (your kaggle API key)
→ Click "Deploy"

Step 4: Wait for Training (50-70 min)
───────────────────────────────────────
• Minutes 0-5: Render builds environment
• Minutes 5-15: Installs Python packages
• Minutes 15-30: Downloads Kaggle dataset
• Minutes 30-70: Trains ResNet50 model
• Minute 70+: API ready to use!

Step 5: Use Your App (Always Available)

→ Open: https://your-app-name.onrender.com

→ Get instant diagnosis & treatment tips

==============================================================
📊 WHAT'S HAPPENING BEHIND THE SCENES
==============================================================

Training Phase (Runs Once):
  1. Download banana leaf disease dataset from Kaggle (~500MB)
  2. Load 3,500+ images and split into train/validation
  3. Build ResNet50 model (pre-trained on ImageNet)
  4. Train custom classification head (10 epochs)
  5. Evaluate accuracy on validation set
  6. Save trained model (95MB) to disk

Serving Phase (Until Server Restart):
  1. Flask server starts and waits
  2. First API request triggers model load
  3. Model loaded into server memory (2-3 seconds)
  4. Model stays in memory for all future requests
  5. Fast predictions (~100-200ms per image)
  6. No retraining needed between requests
  7. Server restart reloads model from disk

Benefits:
  ✓ Fast predictions (no model loading per request)
  ✓ Efficient resource usage (one copy in memory)
  ✓ Works on free tier (no GPU needed)
  ✓ Scales well with traffic (model cached)

⚙️ CUSTOMIZATION
==============================================================

Change Training Settings:
→ Edit train.py
→ EPOCHS = 5  (faster, less accurate)
→ BATCH_SIZE = 8  (less memory)

Change UI Colors:
→ Edit static/style.css
→ Change --primary-color, --secondary-color, etc.

Change Disease Info:
→ Edit app.py
→ Modify DISEASE_INFO dictionary

Change Prediction Thresholds:
→ Edit app.py
→ CONFIDENCE_THRESHOLD = 0.60  (more permissive)
→ GREEN_RATIO_THRESHOLD = 0.05  (stricter)

Modify Disease Classes:
→ Retrain with custom dataset
→ Update class_names in train.py

==============================================================
🐛 TROUBLESHOOTING
==============================================================

❌ "502 Bad Gateway" on deployed site
→ Model still training. Wait another 10-15 minutes.
→ Check Render logs: Dashboard → Logs tab

❌ Web page loads but predictions don't work
→ Model not loaded yet. Wait 2-3 seconds.
→ Try uploading image again.

❌ Training timeout on Render (>2 hours)
→ Reduce EPOCHS from 10 to 5 in train.py
→ Reduce BATCH_SIZE from 16 to 8

❌ Out of memory error during training
→ Edit train.py, change BATCH_SIZE = 8
→ Or reduce EPOCHS to 5

❌ Kaggle download fails
→ Verify username and API key are correct
→ Try downloading dataset manually on Kaggle

❌ Predictions very slow (>5 seconds)
→ Model running on free tier shared CPU
→ Upgrade to Render paid plan ($7+/month)
